# 放映（present）

slides 做完之后，**直接在浏览器里全屏放映**——这是「放映用的网页
slides」，不是要导出成视频，所以**没有音频 / 录屏环节**。

---

## 起服务 + 全屏

1. `npm run dev` 起本地服务，浏览器打开页面（默认 `http://localhost:5173`）
2. **浏览器全屏**（F11 / 浏览器自带的全屏菜单），letterbox 自动把
   1920×1080 舞台缩到当前屏，四周留黑
3. 点击舞台空白处或按方向键推进 step

---

## 键盘导航

| 按键 | 行为 |
|---|---|
| `←` / `→` / `空格` | 上一步 / 下一步推进 step |
| `Home` / `End` | 跳到最前 / 最后 |
| 数字键 `1`–`9` | 跳到第 N 章 |
| **`P`** | **开「演讲者窗口」（presenter view）—— 投屏时看口播稿的正确方式** |
| **`N`** | **开 / 关备注浮层（叠在当前窗口上，仅适合一人对屏排练）** |

---

## ⚠️ 关键区别：备注浮层（N） vs 演讲者窗口（P）

这两个都能让你看到 `notes.ts` 里的口播稿，但**投屏行为完全不同**：

### 备注浮层（按 `N`）—— 排练用，会被投屏

按 `N` 是在**当前这一个窗口、同一个 DOM 上**叠一层浮层。它和 slide
**在同一个窗口里**，所以：

- 共享整个屏幕 → 一开浮层，观众就看到口播稿。
- 哪怕只共享这个窗口 / 这个标签页 → 浮层本来就在这个窗口里，**照样被 share 出去**。

> 所以备注浮层**只适合一个人对着屏幕排练**，不要在投屏 / 录屏时打开。

### 演讲者窗口（按 `P`）—— 投屏用，观众看不到

按 `P` 会 `window.open` 一个**独立的浏览器窗口**（`?presenter`），里面是一个
演讲者控制台：

- 当前页的**实时缩略图**（和观众看到的完全一致，含逐步揭示状态）
- **当前这一步的口播稿**（大字、可滚动）
- **下一步 / 下一章的口播稿**预览
- **计时器**（点一下或按 `R` 归零）+ 页码 / 全局进度

两个窗口通过 `BroadcastChannel` **自动联动**：在任意一个窗口翻页，另一个跟着走。
口播稿**只在演讲者窗口里**，永远不进主 slide 窗口。

---

## 投屏 / 会议（Meet / Zoom / GVC）——看不到口播稿的正确姿势

关键不是「关掉备注」，而是**共享对象选窗口 / 标签页，而不是整个屏幕**：

1. 按 `P` 打开演讲者窗口。
2. 主 slide 窗口按 F11 全屏（或保持普通窗口都行——共享抓的是它的渲染内容）。
3. 在会议软件里选 **「共享某个窗口 / 标签页（a window / a tab）」**，
   选中**主 slide 窗口**。**不要选「共享整个屏幕」**。
4. 把演讲者窗口留在自己面前。观众只看到干净的 slide；翻页两边联动。

> **单屏也成立**：会议软件抓的是「主 slide 窗口」那个表面的渲染内容，
> 跟它在你屏幕上多大、是否被演讲者窗口挡住无关。所以你完全可以把主窗口
> 缩小放角落、演讲者窗口铺满自己这块屏——观众看到的依然是完整的 slide。
> 双屏当然更舒服（一块全屏放 slide，一块放演讲者窗口），但不是必需。
>
> **唯一的雷区**：选了「共享整个屏幕」。那样什么都藏不住——演讲者窗口
> 也会一起进画面。务必选窗口 / 标签页级共享。

## 演讲备注从哪来

每章的提词来自该章 `notes.ts` 里 `export const notes: StepNote[]`——
一个 step 对应数组里一条。这是**给演讲者的提词**（喂演讲者窗口 P 和
备注浮层 N），**不会渲染到 slide 上**，也不参与任何音频 / 视频流程（本
Skill 没有这些）。`notes.length` 同时是该章 step 数的唯一真相源（见
[`CHAPTER-CRAFT.md`](CHAPTER-CRAFT.md)）。

## 现场 / 离线

- **现场演示**：本地 `npm run dev` 跑着就能放，最省事。
- **离线分发 / 换台机器讲**：`npm run build` 出静态产物后，用
  `npm run preview` 本地预览，或直接把 `dist/` 部署 / 拷走。演讲者窗口
  （`?presenter`）在 build 产物里同样可用。

---

## 可选导出（分享 / 留档用）

需要把 slides 变成可静态分享的东西时：

| 方式 | 怎么做 | 注意 |
|---|---|---|
| **逐步截图** | 每推进一个 step 截一张 PNG | 最稳——能精确留住每一 step 的画面状态 |
| **打印为 PDF** | 浏览器「打印 → 另存为 PDF」逐步导出 | **逐 step 的动画状态不一定能完整导出**（动画进行中的中间态、Canvas / 持续微动尤其容易丢） |

> 因此：**优先现场放映或逐步截图**；PDF 适合「静态版讲义」而不是
> 「还原动画」。需要动画效果就现场放映。

---

## 一句话

这是**放映用的网页 slides**（点击驱动、16:9、全屏、键盘导航）——按 `P`
开独立的**演讲者窗口**（口播稿 + 实时预览 + 计时器，两窗联动），投屏时
**只共享主 slide 窗口**，观众就看不到口播稿；按 `N` 的备注浮层只适合
一人排练。**不导出视频**，所以没有 TTS / 音频合成 / 录屏这些步骤。
